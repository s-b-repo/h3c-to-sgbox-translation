# Python Code Audit — Bugs & Flaws (v2.0.0)

Audit of all `.py` files in `src/`. Findings are ordered by severity.

**Status: ✅ All 15 bugs listed below have been FIXED and VERIFIED as of the latest commit (v2.0.0).**

---

## 🔴 Critical / High Severity

### 1. Missing `server_hostname` in TLS connect (`syslog_forwarder.py:100-104`)
`ssl.PROTOCOL_TLS_CLIENT` defaults to `check_hostname=True`, but `asyncio.open_connection()` does not pass `server_hostname` when connecting via IP. 
- **Impact**: TLS handshake to SGBox will fail if `protocol = tls` is used with an IP address instead of a domain name.
- **Fix**: Set `server_hostname=self.host` if using a domain name, or set `ctx.check_hostname = False` with `ctx.verify_mode = ssl.CERT_REQUIRED` if connecting strictly by IP.

### 2. Race condition on stats (`formatter.py:107`)
The `skipped` counter is incremented **outside** the lock on line 107, while other stat updates are inside the lock.
- **Impact**: Data race under concurrent access.
- **Fix**: Wrap the increment in `with self._stats_lock:`.

### 3. Rate limiter memory leak (`api_server.py:52`)
`_RateLimiter._hits` grows unboundedly. Old IPs are never evicted — only their timestamps are pruned per-IP when that IP makes a *new* request.
- **Impact**: An attacker sending one request per IP from a botnet fills this dict forever, causing an Out-Of-Memory (OOM) crash over time.
- **Fix**: Add a periodic cleanup task or use an LRU cache with a maximum size.

## 🟡 Medium Severity

### 4. `threading.Lock` in async context (`parser.py:97`, `formatter.py:77`)
Both `H3CLogParser` and `SGBoxFormatter` use `threading.Lock` for stats.
- **Impact**: `threading.Lock` blocks the entire thread, violating `asyncio` principles. While functionally safe in a single-threaded event loop, it introduces subtle stalls.
- **Fix**: Replace with `asyncio.Lock()`, or remove locks entirely (as simple `+=` is mostly safe in single-threaded asyncio).

### 5. Semaphore internal state access (`syslog_receiver.py:123`)
Accesses `self._connection_semaphore._value`.
- **Impact**: Breaking encapsulation. This is a private implementation detail of `asyncio.Semaphore` that could change in future Python versions.
- **Fix**: Track the connection count in a separate integer field.

### 6. Log file handler not connected to structlog (`translator.py:115`)
`setup_structlog()` configures structlog to print to stdout. It separately creates a stdlib `RotatingFileHandler`. 
- **Impact**: Application logs generated by structlog will **not** appear in the configured log file; only errors from stdlib libraries (like `aiohttp`) will.
- **Fix**: Add `structlog.stdlib.ProcessorFormatter` to bridge structlog events into the stdlib `logging` handler chain.

### 7. No body size check on single translate (`api_server.py:261`)
The `_handle_translate` endpoint reads the body via `request.text()` without a size limit.
- **Impact**: Even though `client_max_size=10MB` is set on the app, a 10MB single translate request could still cause memory pressure, whereas bulk translate caps at 1000 lines.
- **Fix**: Add a strict string length check before parsing.

### 8. `request.remote` doesn't work behind reverse proxy (`api_server.py:179,210`)
`request.remote` returns the direct TCP peer (e.g., the load balancer IP).
- **Impact**: All rate limiting and IP whitelisting apply to the proxy instead of the real client.
- **Fix**: Conditionally parse the `X-Forwarded-For` header based on a trusted proxy IP whitelist.

### 9. File handle hygiene in daemonize (`translator.py:174-180`)
After `dup2()`, the original Python file objects (`devnull_r`, `devnull_w`) are `.close()`d.
- **Impact**: While technically fine on Linux, relying on this behavior is fragile because `dup2` already closed the old FD.
- **Fix**: Use `os.open(os.devnull, os.O_RDWR)` instead of Python's `open()`.

## 🟢 Low Severity / Code Quality

### 10. Dead code in parser cleanup (`parser.py:172-177`)
A loop iterates over all parsed keys but executes `pass`, consuming CPU cycles unnecessarily.

### 11. CORS origin set to `"null"` (`api_server.py:232`)
`Access-Control-Allow-Origin: "null"` is a valid CORS origin for sandboxed iframes. It should be removed entirely for a backend-only REST API to prevent unintended access from local `.html` files.

### 12. Imports inside functions (`syslog_forwarder.py:182`, `api_server.py:276`)
`import os` and `import json` are done inside method/function bodies. Move to the top of the file for consistency and to avoid overhead.

### 13. CSV parsed hostname is discarded (`parser.py:205-213`)
When parsing CSV input, `parts[1]` (the firewall hostname) is extracted by the regex but completely ignored/discarded before translation.

### 14. Inconsistent writer cleanup (`syslog_output_server.py:111`)
When an SGBox collector disconnects, only `Exception` is caught during writer cleanup. It's safe but inconsistent with `syslog_forwarder.py` which uses a dedicated `_safe_close` pattern catching `ConnectionResetError`.

### 15. No `__all__` export (`__init__.py`)
`src/__init__.py` lacks an `__all__` export declaration, which is a good practice for defining the public interface of the package.
